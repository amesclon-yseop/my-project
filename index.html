Yseop Savvy API ============== ​ **Yseop Savvy API** provides some tools to understand graphs, diagrams and charts, by generating clear and intelligible descriptions. Those texts are not technical, they simply highlight the major graph variations, without trying to speculate on the reasons for these changes. This API is mainly meant for BI purposes but can be used in various sectors, as it is flexible and not semantically bound to a specific domain. ​ For now, it already has been integrated in **Yseop Savvy for Qlik** (plugin for Qlik Sense Desktop), and **Yseop Savvy for Microsoft Excel** (add-in for Microsoft Office 2007 and further). ---------- ​ Plan: ​ [TOC] ​ ---------- ​ General Functioning ------------- ​ Yseop Savvy API is a **REST API**[^restapi] . ​ Header Parameters ------------- ​ ### Security ​ The API needs a **Basic Authentication**. The *login* is the JWT token you've been given on [Yseop Savvy's Website](https://savvy.yseop.com/download/). If you don't have any yet, please go get one. The *password* must stay empty. ​ ``` Authorisation="Basic myKey:" ``` ​ ### Input Type ​ You need to indicate the input format, which is always *json*: ``` Content-Type="application/json;charset=UTF-8" ``` ​ ### Output Format ​ The output format has to be set as *html*: ``` Accept="text/html" ``` ​ Chart Description ------------- ​ ##### TOP LEVEL MODEL ​ Top-level object that give all the information to the application. ​ Field | Type| Description -------- | ---| ------ extensionVersion| String| Version Number of the input. charts| [[ChartObject](#chart-object)]| List of charts to describe. ​ ``` { "extensionVersion": "2", "charts": [ ... ] } ``` ​ ##### CHART OBJECT ​ Chart Information. ​ Field | Type| Description -------- | ---| ------ inputChart| [InputChartObject](#input-chart-object)| General information on the Chart. outputText| [OutputParamObject](#output-param-object)| Meta-data for preferences relating to the output text. dimensions| [[DimensionInfoObject](#dimension-info-object)]| There must be one Dimension block per dimension in the chart, 2 max. measures| [[MeasureInfoObject](#measure-info-object)]| There must be one Measure block per measure in the chart, 2 max. facts| [[FactObject](#fact-object)]| List of facts, representing the measures values for each dimension (or couple of dimensions). ​ ​ ##### INPUT CHART OBJECT ​ Field | Type| Description -------- | ---| ------ type| [ChartTypeEnum](#chart-type-enum)| Format of the chart (choose among *Line chart*, *Bar chart* or *Pie chart*). mainDimensionIndex| Integer| The index of the dimension which is considered as 'main' in the analysis. Only used when there are several dimensions. mainDimensionOrdered| Boolean| Deprecated. Use the property *ordered* of each dimension description. multiSeriesType| [[MultiSeriesEnum](#multi-series-enum)]| How the data are related to each other (*Unrelated*, or *Value vs Reference*). disableInterpolation| Boolean | TODO currentSelection| [[SelectionObject](#fact-object)]| TODO otherPartPresents| Boolean| TODO singularExceptions| [String]| TODO pluralExceptions| [String]| TODO ​ ``` { "type": "LINE_CHART", "mainDimensionIndex": 0, "multiSeriesType": "UNRELATED" } ``` ​ ##### OUTPUT PARAM OBJECT ​ Field | Type| Description -------- | ---| ------ levelOfDetail| Integer| Number between *1* and *7*, where 1 produces the shortest text, and 7 the most verbose one. lang| [LangEnum](#lang-enum)| Language of the generated text. ​ ``` { "levelOfDetail":7, "lang":"EN" } ``` ​ ##### DIMENSION INFO OBJECT ​ Field | Type| Description -------- | ---| ------ label| String | The Dimension's name (can contain spaces). othersLabel| String| Sometimes data are grouped in a section 'Other' in graphs. You can define here a specific label for this section. cardinal| Integer| Number of elements in the Dimension. ​ ``` { "label": "month", "othersLabel": "Autres", "cardinal": 12 } ``` ​ ##### MEASURE INFO OBJECT ​ Field | Type| Description -------- | ---| ------ label| String | The Dimension's name (can contain spaces). technicalLabel| Boolean| Sometimes data are grouped in a section 'Other' in graphs. You can define here a specific label for this section. defaultValue| String| Number of elements in the Dimension. meaningOfUp| [MeaningOfUpEnum](#meaning-of-up-enum)| Sometimes data are grouped in a section 'Other' in graphs. You can define here a specific label for this section. unit| String| Number of elements in the Dimension. ​ ``` { "label": "sales", "technicalLabel": false, "defaultValue": "0", "meaningOfUp": "GOOD", "unit": "$" } ``` ​ ##### FACT OBJECT ​ Field | Type| Description -------- | ---| ------ dimensions| [[MeasureValueObject](#measure-value-object)]| TODO measures| [[DimensionValueObject](#dimension-value-object)]| TODO ​ ``` { "dimensions":[{ "index": 0, "label": "01/01/2015", "position": 0 }], "measures":[{ "index": 0, "value": "150" }] } ``` ​ ##### SELECTION OBJECT ​ ##### DIMENSION VALUE OBJECT ​ Field | Type| Description -------- | ---| ------ index| Integer| Index of the dimension in the 'Dimensions' array (indexes start at 0). label| String | Used in text generation for identifying this point. position| Integer| The position is the 'coordinate' of the dimension point related to other points on this dimension. For instance January is the first month. Only relevant for ordered dimensions. ​ ``` { "index": 0, "label": "01/01/2015", "position": 0 } ``` ​ ##### MEASURE VALUE OBJECT ​ ​ Field | Type| Description -------- | ---| ------ index| Integer| Index of the measure in the 'Measures' array (indexes start at 0). value| String | The value for this measure (only numeric values are accepted). ​ ``` { "index": 0, "value": "150" } ``` ​ ​ ##### CHART TYPE ENUM ​ Value| Description ---- | ------ LINE_CHART| Use to describe line or points charts.
